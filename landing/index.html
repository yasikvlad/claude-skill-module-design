// Зачем

Все строят пайплайны
на ощупь.

А потом — две недели рефактора, потому что «этот класс делает всё подряд», «непонятно где менять конфиг» и «давайте просто перепишем».

Скилл превращает «как-нибудь» в жёсткий процесс. На выходе — готовый design blueprint, который не стыдно показать команде.

🌀

// Workflow

9 шагов до чертежа

Каждый шаг закрывает конкретное архитектурное решение. Пропустишь — пробьёт через месяц.

STEP 01

ЦКП системы

Одно предложение. Конкретный продукт, не процесс. Без чего система мертва — определяешь сразу.

STEP 02

Инверсия

Премортем, необходимые условия, bottleneck, kill criteria. До декомпозиции, не после — пока ресурсы не вложены.

STEP 03

Модули

Один заказчик — одна граница. Контракт = вход + выход + ошибка. Меняется по двум причинам — это два модуля.

STEP 04

Этапы

Тест существительного. Разные слова → граница этапа. Одно слово с прилагательным → не граница.

STEP 05

Операции

Атомарный шаг — один семантический акт. Дальше делить — нерабочие фрагменты.

STEP 06

Триада

Config / Client / Storage. Один владелец на уровне. Не дублируется между модулем и этапом.

STEP 07

Шаблон

Развернуть этап в папку только если есть нагрузка: своя оркестрация, своя триада, нечитаемо в одном файле.

STEP 08

Чек-листы

Модуль / этап / операция. Любой «нет» — единица собрана неверно, перепроектируй.

STEP 09

Инверсионная отчётка

Каждый failure mode из шага 2 закрыт конкретной мерой. Никаких «провели премортем и забыли».

Применимо везде,

Методология фрактальна — работает на масштабе операции, модуля, продукта, бизнеса.

🤖

AI-агенты и пайплайны

Sales-bot, content-pipeline, research-agent. Декомпозируешь логику агента в этапы, понимаешь где сидит модель, где промт, где tools.

🕸️

Парсеры и скрейперы

URL → HTML → структура → база. Каждая смена существительного — отдельный этап. Триада на уровне harvester.

⚙️

ETL и data pipelines

Extract → Transform → Load. Чёткие контракты между этапами, понятно где менять формат, где источник, где приёмник.

🧱

Модульные бэкенды

Auth, billing, notifications. Каждый модуль — чёрный ящик с контрактом. Меняешь Stripe на LemonSqueezy — трогаешь один Client.
