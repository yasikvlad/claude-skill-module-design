// Зачем

Все строят пайплайны
на ощупь.

А потом две недели рефактора, потому что «этот класс делает всё подряд», «непонятно где менять конфиг» и «давайте просто перепишем».

Скилл превращает «как-нибудь» в жёсткий процесс. На выходе дизайн-блюпринт, который не стыдно показать команде.

Границы размыты

Модуль utils делает всё подряд. Меняется по любому поводу. SRP идёт лесом.

Этапы вкладываются

Подэтапы в этапах, иерархия плывёт. Через месяц никто не помнит, где живёт логика.

Конфиг везде

API-ключ дублируется в трёх местах. Client тащит бизнес-логику. Storage не заменишь без боли.

// Workflow

9 шагов до чертежа

Каждый шаг закрывает конкретное архитектурное решение. Пропустишь — пробьёт через месяц.

STEP 01 FOUNDATION

ЦКП системы

Одно предложение. Конкретный продукт, не процесс. Без чего система мертва.

STEP 02 FOUNDATION

Инверсия

Премортем, необходимые условия, bottleneck, kill criteria. До декомпозиции, не после.

STEP 03 DECOMPOSE

Модули

Один заказчик — одна граница. Контракт = вход + выход + ошибка.

STEP 04 DECOMPOSE

Этапы

Тест существительного. Разные слова → граница этапа. Одно слово с прилагательным → не граница.

STEP 05 DECOMPOSE

Операции

Атомарный шаг — один семантический акт. Дальше делить — нерабочие фрагменты.

STEP 06 INFRA

Триада

Config / Client / Storage. Один владелец на уровне. Не дублируется между модулем и этапом.

STEP 07 SHAPE

Шаблон

Развернуть этап в папку только если есть нагрузка: своя оркестрация, своя триада.

STEP 08 VALIDATE

Чек-листы

Модуль / этап / операция. Любой «нет» — единица собрана неверно.

STEP 09 VALIDATE

Инверсионная отчётка

Каждый failure mode из шага 2 закрыт конкретной мерой. Никаких «провели премортем и забыли».

// Где работает

Применимо везде,
где есть модули

Методология фрактальна — работает на масштабе операции, модуля, продукта, бизнеса.

AI-агенты

Sales-bot, content-pipeline, research-agent. Понятно где модель, где промт, где tools.

Парсеры и скрейперы

URL → HTML → структура → база. Каждая смена существительного — отдельный этап.

ETL и data pipelines

Extract → Transform → Load. Чёткие контракты, понятно где менять формат и источник.

Модульные бэкенды

Auth, billing, notifications. Меняешь Stripe на LemonSqueezy — трогаешь один Client.
