# module-design

> Engineering design methodology as a Claude Code skill.
> Лендинг: **https://module-design-skill.vercel.app**

Превращает любую задачу разработки в чертёж: **Модуль → Этап → Операция**, с ЦКП на каждом уровне, инверсией от провала, инфраструктурной триадой Config/Client/Storage и шаблоном файловой структуры.

Никаких «давайте подумаем как разложить». Жёсткий 9-шаговый процесс, который выходит на готовый design blueprint.

## Установка

```bash
git clone https://github.com/yasikvlad/claude-skill-module-design ~/.claude/skills/module-design
```

Запусти Claude Code — скилл активируется автоматически на триггеры.

## Что внутри

| Файл | Что |

| `SKILL.md` | 9-шаговый workflow + правила + common mistakes |

| `references/template.md` | Шаблон модуля: свёрнутая и развёрнутая форма |
| `references/checklists.md` | Чек-листы модуля / этапа / операции с примерами |
| `assets/blueprint.md` | Шаблон финального design document |

## Workflow

1. **ЦКП системы** — одно предложение, конкретный продукт
2. **Инверсия первым делом** — премортем, необходимые условия, kill criteria
3. **Декомпозиция в модули** — один заказчик, один контракт
4. **Декомпозиция в этапы** — тест существительного + расширение для пограничных случаев
5. **Декомпозиция в операции** — атомарный шаг, неделим
6. **Размещение триады Config/Client/Storage** — на правильном уровне
7. **Развернуть или свернуть этап** — по 4 критериям нагрузки
8. **Валидация чек-листами** — модуль/этап/операция
9. **Инверсионная отчётка** — каждый failure mode → конкретная мера

## Триггеры

Скилл активируется на упоминания:
- «спроектируй модуль X», «разложи задачу Y на этапы»
- «это модуль или этап», «нужно ли разворачивать этап»
- «где жить Config/Client/Storage»
- «провести премортем», «найти bottleneck», «определить kill criteria»
- ключевые слова: ЦКП, модуль/этап/операция, SRP, инверсия, шаблон модуля, триада

## Применимо к

- AI-агенты и пайплайны
- Парсеры, ETL, контент-конвейеры
- Бэкенд-системы с модульной архитектурой
- Любая задача, где нужна декомпозиция и контракт
